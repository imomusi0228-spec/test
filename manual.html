VRC Cast Companion

VRChatイベント・店舗用 芳名帳＆キャストメモ連携システム 取扱説明書

1. System Overview (システム概要)

この度は「VRC Cast Companion」をご購入いただきありがとうございます。
本システムは、VRChatワールド内に設置する3D芳名帳と、キャストが手元（ブラウザ・スマホ）で確認する顧客管理Webダッシュボードを、無料のクラウドベース（Supabase / Vercel）でリアルタイムに同期・連携させるアセットパッケージです。

💡 特徴

・ゲストがワールドに入室した瞬間に自動検知。PC起動などの手間は一切なく、自動で来店回数が+1されます。
・入室と同時に、キャストのWeb画面が自動的にそのゲストのカルテページに切り替わります。
・ご自身専用のデータベースを構築するため、情報漏洩や消失の心配がなく、永久無料URLで運用可能です。
・ワールド内名簿には暗証番号（パスコード）ロック機能があり、一般ゲストからのデータ覗き見を防ぎます。

2. セットアップ手順

1 データベース（Supabase）の作成と接続情報の取得

1. Supabase (https://supabase.com) にアクセスし、無料アカウントを作成します。
2. 新しいプロジェクトを作成します（データベースのパスワードは紛失しないよう控えてください）。
3. プロジェクトが作成されたら、左メニューの SQL Editor を開き、New Query をクリックします。
4. 以下のSQL文をすべてコピーして貼り付け、右下の Run ボタンをクリックして実行します。 

-- 1. organizations（店舗・グループ）テーブル作成
CREATE TABLE IF NOT EXISTS public.organizations (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    name text NOT NULL,
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- 2. member_profiles（メンバーアカウント）テーブル作成
CREATE TABLE IF NOT EXISTS public.member_profiles (
    id uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    org_id uuid REFERENCES public.organizations(id) ON DELETE CASCADE NOT NULL,
    display_name text,
    vrc_name text,
    role text NOT NULL CHECK (role IN ('master', 'cast')) DEFAULT 'cast',
    updated_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- 3. books（手帳データベース）テーブル作成
CREATE TABLE IF NOT EXISTS public.books (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    org_id uuid REFERENCES public.organizations(id) ON DELETE CASCADE NOT NULL,
    name text NOT NULL,
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- 4. guests テーブルの移行・修正
ALTER TABLE public.guests ADD COLUMN IF NOT EXISTS book_id uuid REFERENCES public.books(id) ON DELETE CASCADE;

-- [既存データ移行] すでに guests があり、かつ book_id が未設定の場合のマイグレーション処理
DO $$
DECLARE
    default_org_id uuid;
    default_book_id uuid;
BEGIN
    IF EXISTS (SELECT 1 FROM public.guests WHERE book_id IS NULL) THEN
        INSERT INTO public.organizations (name) VALUES ('デフォルト店舗') RETURNING id INTO default_org_id;
        INSERT INTO public.books (org_id, name) VALUES (default_org_id, 'デフォルト手帳') RETURNING id INTO default_book_id;
        UPDATE public.guests SET book_id = default_book_id WHERE book_id IS NULL;
    END IF;
END $$;

-- book_id の必須化
ALTER TABLE public.guests ALTER COLUMN book_id SET NOT NULL;

-- 5. realtime_events テーブルの作成・移行
CREATE TABLE IF NOT EXISTS public.realtime_events (
  id bigint generated always as identity primary key,
  org_id uuid REFERENCES public.organizations(id) ON DELETE CASCADE,
  type text not null,
  player_name text,
  comment text,
  visit_count integer,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);
ALTER TABLE public.realtime_events ADD COLUMN IF NOT EXISTS org_id uuid REFERENCES public.organizations(id) ON DELETE CASCADE;

-- 6. 行レベルセキュリティ（RLS）の有効化
ALTER TABLE public.organizations ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.member_profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.books ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.guests ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.realtime_events ENABLE ROW LEVEL SECURITY;

-- 7. セキュリティヘルパー関数の定義
CREATE OR REPLACE FUNCTION public.get_user_org_id()
RETURNS uuid AS $$
    SELECT org_id FROM public.member_profiles WHERE id = auth.uid();
$$ LANGUAGE sql SECURITY DEFINER;

CREATE OR REPLACE FUNCTION public.get_user_role()
RETURNS text AS $$
    SELECT role FROM public.member_profiles WHERE id = auth.uid();
$$ LANGUAGE sql SECURITY DEFINER;

-- 8. RLSポリシーの定義
-- member_profiles
CREATE POLICY "所属する組織のメンバープロフィールを参照可能" ON public.member_profiles
    FOR SELECT USING (org_id = public.get_user_org_id());

CREATE POLICY "自分自身のプロフィールは更新可能" ON public.member_profiles
    FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "マスターはメンバーを管理可能" ON public.member_profiles
    FOR ALL USING (org_id = public.get_user_org_id() AND public.get_user_role() = 'master');

-- organizations
CREATE POLICY "所属する組織情報のみ参照可能" ON public.organizations
    FOR SELECT USING (id = public.get_user_org_id());

-- books
CREATE POLICY "所属組織の手帳のみ参照可能" ON public.books
    FOR SELECT USING (org_id = public.get_user_org_id());

CREATE POLICY "マスターは手帳を管理可能" ON public.books
    FOR ALL USING (org_id = public.get_user_org_id() AND public.get_user_role() = 'master');

-- guests
CREATE POLICY "所属組織の手帳の顧客のみ参照可能" ON public.guests
    FOR SELECT USING (book_id IN (SELECT id FROM public.books WHERE org_id = public.get_user_org_id()));

CREATE POLICY "所属組織の手帳の顧客データを管理可能" ON public.guests
    FOR ALL USING (book_id IN (SELECT id FROM public.books WHERE org_id = public.get_user_org_id()));

-- realtime_events
CREATE POLICY "所属組織のイベントのみ参照可能" ON public.realtime_events
    FOR SELECT USING (org_id = public.get_user_org_id());

CREATE POLICY "匿名ユーザーからのイベント挿入を許可" ON public.realtime_events
    FOR INSERT WITH CHECK (true);

-- 9. 新規ユーザー作成時に自動で member_profiles を作成する PostgreSQL トリガー
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS trigger AS $$
DECLARE
    target_org_id uuid;
    user_role text;
    org_name_val text;
BEGIN
    user_role := coalesce(new.raw_user_meta_data->>'role', 'master');
    
    IF new.raw_user_meta_data->>'org_id' IS NOT NULL THEN
        target_org_id := (new.raw_user_meta_data->>'org_id')::uuid;
    ELSE
        org_name_val := coalesce(new.raw_user_meta_data->>'org_name', '新規店舗');
        INSERT INTO public.organizations (name) VALUES (org_name_val) RETURNING id INTO target_org_id;
        user_role := 'master';
    END IF;

    INSERT INTO public.member_profiles (id, org_id, display_name, vrc_name, role)
    VALUES (
        new.id,
        target_org_id,
        coalesce(new.raw_user_meta_data->>'display_name', split_part(new.email, '@', 1)),
        coalesce(new.raw_user_meta_data->>'vrc_name', ''),
        user_role
    );
    RETURN new;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE OR REPLACE TRIGGER on_auth_user_created
    AFTER INSERT ON auth.users
    FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();

-- 10. リアルタイム配信（レプリケーション）を有効化
ALTER PUBLICATION supabase_realtime ADD TABLE public.guests;
ALTER PUBLICATION supabase_realtime ADD TABLE public.realtime_events;

🔑 接続情報（Project ID と APIキー）の取得手順

WebダッシュボードやUnity（VRChat）と接続するためのIDとキーをコピーします：

  1. 画面左端にある黒い縦一列のメニューの最下部付近にある ⚙️（Project Settings / 歯車マーク） をクリックします。
  2. プロジェクトURL（Project URL）のコピー：
    歯車マークをクリックすると右側に現れるメニューの一番上にある General をクリックします。
    画面中央の General Settings 欄にある Project URL（または API Endpoint）の右側にある [Copy]（📋） ボタンをクリックしてコピーします。
  3. APIキー（anon public key）のコピー：
    同じメニュー内にある API Keys をクリックします。
    画面中央の Publishable key 欄にある default の右側（sb.publishable_...）の [Copy]（📋） ボタンをクリックしてコピーします。
    ※注意：下にある「Secret keys」は管理者用の秘密鍵ですので、絶対に使用しないでください。
2 Webアプリのクラウド公開（Vercelデプロイ）

サーバーやプログラムの知識がなくても、以下のいずれかの方法で簡単にインターネット上へ自分専用の名簿ダッシュボードを公開（デプロイ）できます。どちらも完全無料で利用可能です。

【方法A】1クリックでデプロイ（最も簡単・おすすめ）

以下のボタンをクリックすると、GitHubへのコピー作成からVercelへの公開までをほぼ自動で行えます。

1. 下記の「Deploy to Vercel」ボタンをクリックし、画面の指示に従ってGitHubおよびVercelの無料アカウントを作成・ログインします。
2. リポジトリ作成画面が出たら、任意の名前（例: vrc-cast-companion）を入力して「Create」を押します。
3. 途中で**「Configure Project（プロジェクト設定）」**の画面が表示されたら、Environment Variables（環境変数）の入力欄に、Step 1で取得したSupabaseの値を入力します：

  • SUPABASE_URL : Supabaseの Project URL（例：https://[あなたのProject ID].supabase.co）を入力
  • SUPABASE_KEY : Supabaseの anon public API Key を入力

4. 「Deploy」ボタンを押すと、数十秒で公開URL（例：https://your-name.vercel.app）が完成します！

Deploy to Vercel

【方法B】手動でGitHubにアップロードしてデプロイ

1. 無料の GitHub アカウントを作成し、新規リポジトリを作成します。
2. 本製品の 2_WebDashboard/ フォルダ内のすべてのファイルを、作成したGitHubリポジトリにアップロード（プッシュ）します。（※VSCodeのGit機能や、GitHub Desktopアプリなどを使うとドラッグ＆ドロップ感覚で簡単です）
3. Vercel にログインし、「Add New ➔ Project」から先ほどアップロードしたGitHubリポジトリをインポートします。
4. 設定画面の「Environment Variables」欄に、上記と同様に SUPABASE_URL と SUPABASE_KEY の2つを登録します。
5. 「Deploy」ボタンを押せば、数十秒後に公開URLが生成されます。

3 Unity（VRChatワールド）への導入

1. Unityプロジェクトに 1_UnityAssets/VRCGuestBook.cs をインポートします（UdonSharp Behaviourとして自動変換されます）。
2. ワールド内に名簿UI（本モデル、入力UI、表示UI）を配置し、Udon Behaviourをアタッチします。
3. Udon Behaviour内の変数設定にて、Base Url に Vercelの公開URL/api/join?book_id=手帳ID （例: https://your-app.vercel.app/api/join?book_id=xxxx-xxxx-xxxx-xxxx）を設定します。マルチテナント化に伴い、どの手帳に顧客を登録するかを識別するために、必ずポータル画面で作成した手帳IDを ?book_id=xxx の形式でURLの末尾に付与してください。
4. 以下のセキュリティおよびパネル変数を設定します：

  • Passcode : キャスト閲覧ページを開くための任意の暗証番号（数字4桁など）を入力。
  • Guest Panel : 一般ゲストがコメントを入力するパネルGameObjectをアタッチ。
  • Lock Panel : パスコードを入力するロック画面GameObjectをアタッチ。
  • Cast Panel : ロック解除後に表示されるキャスト用の名簿表示GameObjectをアタッチ。
  • Passcode Input Field : 暗証番号を入力するInputFieldUIをアタッチ。

5. 各種ボタンの OnClick イベントにUdonの SendCustomEvent を登録します：

  • コメント送信ボタン : OnRegisterClick
  • ロック解除ボタン : OnUnlockClick
  • 再びロックするボタン : OnLockClick

3. 動作仕様

自動来店カウント（12時間重複防止）

ゲストがワールドに入室した時点で、Udonが自動的にVercel APIを経由してデータベースへ入室通知を送ります。
この際、前回の入室日時から12時間以上経過していた場合、新しい「来店」とみなして来店カウントが自動的に `+1` されます。
12時間以内の再入室（回線落ちによる再接続など）のときは、来店数はカウントアップされずにログイン情報のみが記録されます。

一般ゲストとキャスト画面の保護

ゲスト用画面には個人情報（他プレイヤーの名前や来店回数の履歴）は一切表示されず、自身のコメント送信欄のみが提供されます。
事前に設定した暗証番号を入力してロックを解除することで、初めてキャスト用パネル（過去の記名履歴・来店回数一覧）が表示されます。これにより、ワールド内でのプライバシーが厳重に守られます。
