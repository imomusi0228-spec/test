VRC Cast Companion

VRChatイベント・店舗用 芳名帳＆キャストメモ連携システム 取扱説明書

1. System Overview (システム概要)

この度は「VRC Cast Companion」をご購入いただきありがとうございます。
本システムは、VRChatワールド内に設置する3D芳名帳と、キャストが手元（ブラウザ・スマホ）で確認する顧客管理Webダッシュボードを、無料のクラウドベース（Supabase / Vercel）でリアルタイムに同期・連携させるアセットパッケージです。

💡 特徴

・ゲストがワールドに入室した瞬間に自動検知。PC起動などの手間は一切なく、自動で来店回数が+1されます。
・入室と同時に、キャストのWeb画面が自動的にそのゲストのカルテページに切り替わります。
・ご自身専用のデータベースを構築するため、情報漏洩や消失の心配がなく、永久無料URLで運用可能です。
・ワールド内名簿には暗証番号（パスコード）ロック機能があり、一般ゲストからのデータ覗き見を防ぎます。

2. セットアップ手順

1 データベース（Supabase）の作成と接続情報の取得

1. Supabase (https://supabase.com) にアクセスし、無料アカウントを作成します。
2. 新しいプロジェクトを作成します（データベースのパスワードは紛失しないよう控えてください）。
3. プロジェクトが作成されたら、左メニューの SQL Editor を開き、New Query をクリックします。
4. 以下のSQL文をすべてコピーして貼り付け、右下の Run ボタンをクリックして実行します。 

-- guests（顧客台帳）テーブルの作成
create table if not exists guests (
  id text primary key,
  name text not null,
  pronunciation text,
  vrc_name text,
  x_id text,
  discord_id text,
  first_visit text,
  last_visit text,
  visit_count integer default 1,
  is_cast boolean default false,
  tags text[] default '{}',
  characteristics text,
  notes jsonb default '[]'::jsonb
);

-- 既存のデータベースがある場合、キャスト用カラムが存在しなければ追加する
alter table guests add column if not exists is_cast boolean default false;

-- realtime_events（入室・記名通知）テーブルの作成
create table if not exists realtime_events (
  id bigint generated always as identity primary key,
  type text not null,
  player_name text,
  comment text,
  visit_count integer,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- リアルタイム配信（レプリケーション）を有効化
alter publication supabase_realtime add table guests;
alter publication supabase_realtime add table realtime_events;

🔑 接続情報（Project ID と APIキー）の取得手順

WebダッシュボードやUnity（VRChat）と接続するためのIDとキーをコピーします：

  1. 画面左端にある黒い縦一列のメニューの最下部付近にある ⚙️（Project Settings / 歯車マーク） をクリックします。
  2. プロジェクトURL（Project URL）のコピー：
    歯車マークをクリックすると右側に現れるメニューの一番上にある General をクリックします。
    画面中央の General Settings 欄にある Project URL（または API Endpoint）の右側にある [Copy]（📋） ボタンをクリックしてコピーします。
  3. APIキー（anon public key）のコピー：
    同じメニュー内にある API Keys をクリックします。
    画面中央の Publishable key 欄にある default の右側（sb.publishable_...）の [Copy]（📋） ボタンをクリックしてコピーします。
    ※注意：下にある「Secret keys」は管理者用の秘密鍵ですので、絶対に使用しないでください。
2 Webアプリのクラウド公開（Vercelデプロイ）

サーバーやプログラムの知識がなくても、以下のいずれかの方法で簡単にインターネット上へ自分専用の名簿ダッシュボードを公開（デプロイ）できます。どちらも完全無料で利用可能です。

【方法A】1クリックでデプロイ（最も簡単・おすすめ）

以下のボタンをクリックすると、GitHubへのコピー作成からVercelへの公開までをほぼ自動で行えます。

1. 下記の「Deploy to Vercel」ボタンをクリックし、画面の指示に従ってGitHubおよびVercelの無料アカウントを作成・ログインします。
2. リポジトリ作成画面が出たら、任意の名前（例: vrc-cast-companion）を入力して「Create」を押します。
3. 途中で**「Configure Project（プロジェクト設定）」**の画面が表示されたら、Environment Variables（環境変数）の入力欄に、Step 1で取得したSupabaseの値を入力します：

  • SUPABASE_URL : Supabaseの Project URL（例：https://[あなたのProject ID].supabase.co）を入力
  • SUPABASE_KEY : Supabaseの anon public API Key を入力

4. 「Deploy」ボタンを押すと、数十秒で公開URL（例：https://your-name.vercel.app）が完成します！

Deploy to Vercel

【方法B】手動でGitHubにアップロードしてデプロイ

1. 無料の GitHub アカウントを作成し、新規リポジトリを作成します。
2. 本製品の 2_WebDashboard/ フォルダ内のすべてのファイルを、作成したGitHubリポジトリにアップロード（プッシュ）します。（※VSCodeのGit機能や、GitHub Desktopアプリなどを使うとドラッグ＆ドロップ感覚で簡単です）
3. Vercel にログインし、「Add New ➔ Project」から先ほどアップロードしたGitHubリポジトリをインポートします。
4. 設定画面の「Environment Variables」欄に、上記と同様に SUPABASE_URL と SUPABASE_KEY の2つを登録します。
5. 「Deploy」ボタンを押せば、数十秒後に公開URLが生成されます。

3 Unity（VRChatワールド）への導入

1. Unityプロジェクトに 1_UnityAssets/VRCGuestBook.cs をインポートします（UdonSharp Behaviourとして自動変換されます）。
2. ワールド内に名簿UI（本モデル、入力UI、表示UI）を配置し、Udon Behaviourをアタッチします。
3. Udon Behaviour内の変数設定にて、Base Url に Vercelの公開URL/api（例: https://your-app.vercel.app/api）を設定します。
4. 以下のセキュリティおよびパネル変数を設定します：

  • Passcode : キャスト閲覧ページを開くための任意の暗証番号（数字4桁など）を入力。
  • Guest Panel : 一般ゲストがコメントを入力するパネルGameObjectをアタッチ。
  • Lock Panel : パスコードを入力するロック画面GameObjectをアタッチ。
  • Cast Panel : ロック解除後に表示されるキャスト用の名簿表示GameObjectをアタッチ。
  • Passcode Input Field : 暗証番号を入力するInputFieldUIをアタッチ。

5. 各種ボタンの OnClick イベントにUdonの SendCustomEvent を登録します：

  • コメント送信ボタン : OnRegisterClick
  • ロック解除ボタン : OnUnlockClick
  • 再びロックするボタン : OnLockClick

3. 動作仕様

自動来店カウント（12時間重複防止）

ゲストがワールドに入室した時点で、Udonが自動的にVercel APIを経由してデータベースへ入室通知を送ります。
この際、前回の入室日時から12時間以上経過していた場合、新しい「来店」とみなして来店カウントが自動的に `+1` されます。
12時間以内の再入室（回線落ちによる再接続など）のときは、来店数はカウントアップされずにログイン情報のみが記録されます。

一般ゲストとキャスト画面の保護

ゲスト用画面には個人情報（他プレイヤーの名前や来店回数の履歴）は一切表示されず、自身のコメント送信欄のみが提供されます。
事前に設定した暗証番号を入力してロックを解除することで、初めてキャスト用パネル（過去の記名履歴・来店回数一覧）が表示されます。これにより、ワールド内でのプライバシーが厳重に守られます。
